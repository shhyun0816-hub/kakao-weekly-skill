# MVP 위클리 카카오 챗봇 스킬

`miraeassetmvp.imweb.me/weekly`의 최신 글 5개를 카카오 i 오픈빌더 "시나리오 05" 블록에
**실시간으로** 리스트카드로 보여주는 스킬 서버입니다. (이미지 없음, 제목 노출, 글마다 바로가기 링크)

---

## 1. Vercel로 배포하기

### 방법 A. Vercel 웹사이트에서 배포 (가장 쉬움)
1. 이 폴더를 GitHub 저장소로 올리기 (또는 zip 그대로 Vercel에 드래그 앤 드롭도 가능)
2. https://vercel.com 가입 후 "Add New… → Project" → 이 저장소 선택
3. Framework Preset: **Other** 선택 (별도 설정 없이 그대로 Deploy)
4. 배포 완료 후 나오는 도메인 확인, 예: `https://kakao-weekly-skill.vercel.app`
5. 실제 스킬 URL은 `https://kakao-weekly-skill.vercel.app/api/skill` 입니다.

### 방법 B. Vercel CLI로 배포
```bash
npm install -g vercel
cd kakao-weekly-skill
npm install
vercel        # 최초 1회 로그인 및 프로젝트 연결
vercel --prod # 프로덕션 배포
```

### 배포 후 확인
브라우저나 curl로 아래 주소에 접속했을 때 JSON이 정상적으로 나오면 성공입니다.
```bash
curl https://<your-domain>/api/skill
```

## 2. 카카오 i 오픈빌더에 스킬 등록하기

1. `i.kakao.com` → **디자센** 봇 → 좌측 메뉴 **스킬 → 스킬 목록**
2. **스킬 만들기** 클릭
   - 스킬명: `MVP위클리최신글`
   - URL: `https://<your-domain>/api/skill`
3. 저장 후, **시나리오 → 시나리오 05 → 블록 추가**
   - 블록명: `MVP_위클리_최신글5개`
   - 사용자 발화 예시 등록: "주간 시황", "위클리 브리핑", "이번 주 시황 알려줘" 등
   - **봇 응답 → 스킬 데이터 사용** 체크 → 방금 만든 `MVP위클리최신글` 스킬 선택
4. 저장 후 상단 **봇테스트**에서 "주간 시황" 등을 입력해 정상 동작 확인
5. 문제 없으면 **배포** 메뉴에서 실제 배포

이제 매번 이 블록이 실행될 때마다 스킬 서버가 weekly 페이지를 새로 확인해서
**항상 최신 글 5개**를 자동으로 보여줍니다. 새 글이 올라와도 오픈빌더를 다시 손댈 필요가 없습니다.

---

## 3. 알아두어야 할 제약사항

- **응답 시간**: 카카오 스킬은 5초 안에 응답해야 합니다. 코드에 5분 캐시를 넣어뒀지만,
  서버리스 특성상 완벽한 캐시는 아닙니다. 트래픽이 많다면 Vercel KV나 Upstash Redis 같은
  외부 캐시 저장소를 추가하는 걸 권장합니다.
- **사이트 구조 변경**: `api/skill.js`의 스크래핑 로직은 imweb.me의 현재 마크업(게시글 링크가
  `/숫자` 형태)을 기준으로 작성했습니다. 사이트 개편 시 셀렉터를 다시 확인해야 할 수 있습니다.
- **봇 차단 가능성**: imweb.me가 서버 요청을 봇으로 인식해 차단할 가능성이 있습니다.
  이 경우 아래 "대안" 섹션을 참고하세요.
- **이미지 없음 / 요약 없음**: 현재는 제목만 노출합니다. 이미지나 자동 요약이 필요해지면
  `buildListCardResponse` 함수에 `imageUrl`, 요약 로직을 추가하면 됩니다.

## 4. 대안: 스크래핑이 막힐 경우

`axios`로 직접 요청했을 때 imweb.me가 403을 반환하며 막는 경우:

1. **헤드리스 브라우저 사용**: `@sparticuz/chromium` + `puppeteer-core` 조합으로
   실제 브라우저처럼 페이지를 렌더링해서 가져오는 방법 (Vercel 서버리스에서도 동작 가능하나
   콜드 스타트가 느려질 수 있음).
2. **스크래핑 프록시 서비스** (ScraperAPI, Browserless 등) 경유.
3. **정기 배치 + 캐시 저장소**: Vercel Cron Job(예: 1시간마다)으로 미리 스크래핑해서
   Vercel KV / Upstash Redis 같은 곳에 저장해두고, 스킬 함수는 그 캐시만 읽어오는 방식.
   이 방식이 가장 안정적이고 응답 속도도 빠릅니다. 필요하시면 이 구조로도 코드를
   다시 작성해드릴 수 있어요.
